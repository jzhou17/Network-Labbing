Lab 1

Date: 3/5/2025

Tools used:
Aruba Central, PuTTY SSH Client

Gear used:
Aruba 9004 Gateway, AOS-CX 6100 Switch

Goal:
Document which VLANs belong to which port on the AOS-CX 6100 switch and the Aruba 9004 gateway. Write down what infrastructure devices are on what ports via LLDP. The use of trace or looking at the physical wire connections is not permitted.

Part 1: Determine VLAN to Port Mappings

AOS-CX 6100 Switch

Determining the VLAN to Port mappings on the AOS-CX 6100 switch was fairly simple. I ran the following command:

6100# show vlan

VLAN to Port Mapping
VLAN Port
1 1/1/15-1/1/16
10 1/1/12-1/1/14
20 1/1/2-1/1/4, 1/1/7-1/1/8, 1/1/12
30 1/1/1-1/1/2, 1/1/4-1/1/6, 1/1/8, 1/1/10-1/1/12
111 1/1/4, 1/1/7-1/1/9

Aruba 9004 Gateway

Likewise, for the Aruba 9004 Gateway I ran the same command:

(Aruba9004)# show vlan

VLAN to Port Mapping
VLAN Port
1 GE0/0/2 Pc0-7
20 GE0/0/3
30 GE0/0/0
111 GE0/0/1 GE0/0/3

Part 2: Determine Device to Port Mappings via LLDP

AOS-CX 6100 Switch

Determining the Device to Port mappings via LLDP on the AOS-CX 6100 switch was fairly simple. I ran the following command:

6100# show lldp neighbor-info

Device to Port Mapping
Port Device Port Description
1/1/1 Aruba9004 (Aruba 9004 Gateway) GE0/0/0
1/1/2 Bayonet ZAG3KH40H1 (AT&T Modem) eth0
1/1/6 eero (Controller?) eth1
1/1/7 Aruba9004 (Aruba 9004 Gateway) LAN
1/1/8 Aruba AP-325 eth0

Aruba 9004 Gateway

Likewise, for the Aruba 9004 Gateway I ran the following command:

(Aruba9004)# show lldp neighbor

Device to Port Mapping
Port Device Remote Interface
GE0/0/0 6100 (AOS-CX 6100 Switch) 1/1/1
GE0/0/3 6100 (AOS-CX 6100 Switch) 1/1/7

Side Takeaway Notes:

  • You cannot SSH into a device over the Internet with its private address. To prevent conflicts and security risks, routers block private IPs from being forwarded onto the public Internet.
